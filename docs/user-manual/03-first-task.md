# 3. 第一次完成任务（从开始任务到拿到结果）

## 文档信息
- 目标读者：已完成模型配置的用户
- 预计耗时：5 分钟
- 成功标准：完成一次“发起任务 -> 继续追问”的完整闭环

## 这页解决什么
教你用最短路径完成第一次任务，建立“可用感”。

## 操作步骤
前提：已完成快速配置向导或在设置里完成模型连接。  
如果未配置模型，页面会继续显示快速配置引导，无法正常开始任务。

1. 点击左侧“开始任务”。
2. 在输入框输入你的任务描述，例如：  
   `请把今天待办分成 P0/P1/P2，并给出可执行计划。`
3. 点击“开始任务”或发送按钮创建会话。
4. 等待首条回复完成。
5. 在同一会话继续追问，例如：  
   `把 P0 拆成今天 30 分钟可执行的 3 个动作。`

## 成功标志
1. 首条消息可正常返回，不报模型错误。
2. 第二次追问能基于上文继续回答。
3. 左侧“会话历史”可看到刚创建的会话。

## 进阶建议
1. 使用更具体的任务模板：目标、限制、输出格式三件套。
2. 会话名称可按“日期 + 主题”命名，方便后续检索。

## 常见问题
1. 点击发送无响应：回到第 2 章检查模型配置。
2. 输出太长或跑题：增加“字数限制”和“输出模板”约束。
3. 误删会话：删除前会有风险确认，确认前请先检查标题。
