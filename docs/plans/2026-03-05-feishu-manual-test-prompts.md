# 飞书手工验收消息模板（多员工协作）

## 用法
- 按顺序复制发送到同一个飞书群。
- 每条发送后，等待桌面端和飞书端都出现预期再发下一条。

## 模板
1. 主员工接管（无 @）
```text
请你们帮我分析一个需求：客户是医药公司，想做一个类似 openclaw + claude cowork 的智能体协作系统。请先给方案框架与报价区间。
```

2. 触发委派与子员工执行
```text
请细化技术方案，重点拆分：架构、MVP范围、里程碑、风险。
```

3. 子员工直达（显式 @）
```text
@开发团队 你细化一下技术方案，给出模块划分和技术栈建议。
```

4. 文本 @ 兜底（同样可用于 role_id 缺失场景）
```text
@开发团队 请继续拆分到 API、任务编排、会话同步、监控告警四层，并标注开发工期。
```

5. 澄清应答（当机器人问你“选 A 还是 B”）
```text
选 A，优先 6 周上线 MVP。
```

6. 汇总追问（检查主员工是否最终汇总）
```text
请项目经理汇总成最终执行方案，包含预算、排期、验收标准。
```
