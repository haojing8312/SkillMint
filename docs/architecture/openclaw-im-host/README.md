# OpenClaw IM Host Historical Notes

> **Historical / superseded folder:** The documents in this directory record the April 2026 OpenClaw-compatible IM host work. They are kept for audit history, regression context, and migration reference.
>
> They are no longer the active architecture target for WorkClaw. The current direction is the Hermes-aligned sidecar removal roadmap: profile runtime boundaries, native Rust `ToolRegistry` / toolsets, platform gateways/adapters, native providers, and profile-owned memory / skills / growth / curator. OpenClaw-shaped names, plugin-host contracts, route aliases, and vendor lanes in these documents should be treated as legacy migration inputs only.
>
> Current source of truth: [Hermes-Aligned Sidecar Removal Roadmap](../../plans/2026-05-11-hermes-aligned-sidecar-removal-roadmap.md) and [OpenClaw Remnant Classification](../../plans/2026-05-11-openclaw-remnant-classification.md).

Do not use this folder as current guidance for new IM, gateway, routing, browser, MCP, toolset, or profile-runtime work. New work should use neutral WorkClaw/Hermes-aligned names and keep any remaining OpenClaw wrappers thin, explicit, and removable.
